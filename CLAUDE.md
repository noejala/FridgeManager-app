## Architecture

**Stack:** React 18 + TypeScript (strict) + Vite. No router — tab switching is handled in `App.tsx` via `activeTab` state. All three tab contents are always mounted (using the `hidden` attribute) to prevent `WhatToCook` from re-fetching on tab switch.

**Auth & data:** Supabase (email/password auth + Postgres). Products are stored in Supabase, not localStorage. Auth state is managed via `supabase.auth.onAuthStateChange` in `App.tsx`. Dark mode preference is still persisted to `localStorage` under `'theme'`.

**State management:** All product state lives in `App.tsx` (no external store). CRUD handlers are async and call `src/utils/productService.ts` directly.

**Data flow:**
- `App.tsx` owns `user`, `products`, `editingProduct`, `activeTab`, `darkMode`, `authLoading`, and `notification` state.
- Renders `<Auth>` when not logged in, the full app when logged in.
- Child components are purely presentational except `WhatToCook`, which fetches from TheMealDB API.

**Three tabs:** `fridge` (ProductList + AddProductForm/EditProductForm), `cook` (WhatToCook), `seasonal` (SeasonalProducts).

## Core Data Model (`src/types/Product.ts`)

```ts
interface Product {
  id: string;                        // UUID generated by Supabase (gen_random_uuid())
  name: string;
  category: ProductCategory;         // 'Fruits'|'Vegetables'|'Meat'|'Fish'|'Dairy'|'Beverages'|'Frozen'|'Other'
  expirationDate: string;            // YYYY-MM-DD
  quantity: number;
  unit: string;                      // 'unit'|'kg'|'L'|...
  addedDate: string;                 // YYYY-MM-DD
  isEstimatedExpiration?: boolean;
  fridgeZone?: string;               // Placement recommendation
}
```

## Supabase

**Table:** `products` — snake_case columns, `user_id` FK to `auth.users`, RLS enabled with `FORCE ROW LEVEL SECURITY`. Index on `user_id`. Policy uses `(select auth.uid())` for performance.

**Env vars** (in `.env.local`, not committed):
```
VITE_SUPABASE_URL=...
VITE_SUPABASE_ANON_KEY=...
```

**Production:** https://fridgemanager.vercel.app — deployed on Vercel, env vars set in Vercel dashboard.

## External API

TheMealDB (no auth required):
- `GET /filter.php?i={ingredient}` — recipes by ingredient
- `GET /lookup.php?i={id}` — full recipe details

## CSS & Design System

Each component has a paired `.css` file (except `EditProductForm`, which shares styles). CSS Grid/Flexbox for layout.

**Design system** — all tokens are CSS custom properties defined in `App.css` under `:root`:
- `--font-display` (Cormorant Garamond) for headings, `--font-body` (Figtree) for everything else
- `--bg`, `--bg-card`, `--bg-surface`, `--bg-input` — background hierarchy (light by default)
- `--text`, `--text-secondary`, `--text-muted` — text hierarchy
- `--accent` (blue `#2c6fe0`) / `--accent-light` / `--accent-bg` for primary actions and highlights
- `--green` / `--amber` / `--red` (+ `*-bg` variants) for product status
- `--border` / `--border-hover` for subtle borders
- `--shadow`, `--radius` for elevation and rounding
- Global keyframes (`card-in`, `fade-in`, `modal-in`, `toast-fade`) defined in `App.css` and used across component CSS files

**Dark mode:** toggled via a button in the header (and on the auth screen). Applies `data-theme="dark"` on `<html>`. All color tokens are overridden in the `[data-theme="dark"]` block in `App.css`.

**Key pattern:** browser autofill on dark inputs requires `-webkit-box-shadow: 0 0 0px 1000px var(--bg-input) inset` and `-webkit-text-fill-color: var(--text)` — applied in `AddProductForm.css` and `Auth.css`.
